DevOps & Infra 知识库

一份知识，两种使用方式 — Web 浏览器阅读，LLM Skill 驱动。搜索不再靠关键词：搜"网站挂了"直接找到"Nginx 故障恢复流程"。

知识库简介

本知识库基于 llm-wiki 框架设计，核心思路是：LLM 负责写入和维护知识，人类通过 Web 阅读知识，Skill 作为连接 LLM 与知识库的桥梁。

🔮

语义搜索

搜"网站挂了"直接找到"Nginx 故障恢复流程"——不需要相同关键词，AI 理解你的意思。

🌐

Web 界面

浏览器中浏览、搜索、上传原始文件，查看含流程图的完整知识内容。

🤖

LLM Skill

在 Claude Code / OpenCode / Copilot 中通过 Skill 导入知识、搜索知识、检查健康状态。

📄

Markdown 本体

知识以 Markdown 文件存储，人和 LLM 都能直接阅读，Git 版本管理。

✅

三级质检

A 级基础 · B 级内容 · C 级全面，确保每条知识可靠、可执行、可追溯。

快速开始

  1. 初始化
    python kb.py init
  2. 配置 LLM API Key（至少一个）
    ANTHROPIC_API_KEY=sk-ant-... 或 DEEPSEEK_API_KEY=sk-... 或 GEMINI_API_KEY=...
  3. 启动 Web 服务
    python kb.py serve → 访问 http://localhost:8000
  4. 在 LLM 中加载 Skill 目录
    将 skills/ 目录配置为 Claude Code / OpenCode / Copilot 的 Skill 来源。

知识页 — 阅读详情

知识详情页从上到下展示：

  1. 元信息区 — ID、分类、版本、状态、作者、更新日期
  2. 背景/场景 — 什么情况下使用这条知识
  3. 操作步骤 — 带命令行的分步指引
  4. 流程图（如果有） — Mermaid 流程可视化
  5. 验证证据 — 检查清单
  6. 回滚方案 — 出问题了怎么恢复
  7. 执行接口 — Harbor CLI、Jenkins、GitHub Action 链接
💡 使用时机： 需要执行某个具体操作时。打开对应知识页，按步骤操作，逐项打勾验证。

管理后台 /admin/ 有两个子页面：

页面功能使用时机

健康页 — 质量报告

访问 /admin/health 查看自动化质量检查报告：

检查级别检查内容不通过后果
A 级 · 基础 必填字段、ID 格式、死链接、Markdown 语法 阻止入库
B 级 · 内容 步骤可执行性、验证证据、回滚方案 阻止入库
C 级 · 全面 知识新鲜度、交叉引用、孤立检测 阻止入库
💡 使用时机： 定期（建议每周）打开检查。发现的问题附带修复指引，逐条处理即可。

LLM 使用 — 工作方式

知识库为 LLM 提供了 3 个元 Skill，它们遵循 agentskill.io 标准，可在 Claude Code、OpenCode、GitHub Copilot 中加载。LLM 通过 Skill → CLI → Engine 的路径操作知识库。

┌──────────┐     ┌──────────────┐     ┌──────────┐
│  用户     │────▶│  LLM + Skill │────▶│  Engine  │
│ "导入这篇"│     │ kb-ingest    │     │ ingest() │
└──────────┘     └──────────────┘     └──────────┘
                                             │
                              ┌──────────────┼──────────────┐
                              ▼              ▼              ▼
                         wiki/*.md     index.md       log.md
                         + .html       (更新)         (追加)
📥

kb-ingest

导入知识：接收文本或文件 → LLM 整理格式 → 去重 → 质检 → 入库 → 生成 Skill

🔍

kb-search

搜索知识：LLM 提取关键词 → Engine 执行搜索 → 返回匹配的知识 Skill

📊

kb-status

健康检查：运行三级质检 → 汇总问题 → 给出修复建议

上传知识 — 使用 kb-ingest

这是将新知识加入知识库的唯一入口。无论通过哪种 LLM 平台，流程一致：

# 在 Copilot Chat 中直接说：
使用 kb-ingest 导入这份文档

[粘贴文档内容或拖入文件]

# 或者指定文件路径：
用 kb-ingest 把 raw/nginx-restart-guide.md 导入知识库
# 在 Claude Code 中：
> /kb-ingest

Claude 会引导你：
1. 提供知识内容（粘贴文本或指定文件路径）
2. LLM 自动按模板整理
3. LLM 检查完整性，缺失项会提示你补充
4. 确认后自动入库
# 在 OpenCode 中：
请使用 kb-ingest skill 把下面的内容导入知识库：

[粘贴内容]

OpenCode 会调用 python kb.py ingest 完成入库。
  1. 提供原始内容 — 粘贴文本或指定 raw/ 目录下的文件路径
  2. LLM 整理 — 自动提取背景、步骤、验证、回滚，补充缺失信息
  3. LLM 写入临时文件 — 将整理好的 Frontmatter + Markdown 保存为临时文件
  4. 调用 CLI 入库 — python kb.py ingest raw/doc.md --content /tmp/llm-output.md
  5. 完整 Pipeline 执行 — 去重 → 质检 → 写入 wiki → 更新索引 → 生成 Skill → 建立向量索引

阅读知识 — 使用 kb-search

搜索是 LLM 阅读知识库的入口。LLM 不需要遍历文件，而是通过 Engine 高效检索。

# 在 Copilot Chat 中：
kb-search nginx 怎么安全重启

# Copilot 会：
# 1. 调用 python kb.py search "nginx 重启" --category infra-daily
# 2. 返回匹配的知识条目列表
# 3. 加载最匹配的知识 Skill
# 4. 展示完整操作步骤
# 在 Claude Code 中：
> /kb-search nginx 重启流程

Claude 会搜索知识库并给出操作指引。
# 在 OpenCode 中：
用 kb-search 查找磁盘清理的操作步骤
Token 节省设计： 搜索时只返回标题 + 摘要（~50 tokens/条）。确认需要的条目后，才加载完整 Skill（~500 tokens）。避免 200 条知识全部读入上下文。

检查状态 — 使用 kb-status

定期让 LLM 检查知识库健康状态，发现并修复问题：

# 任何 LLM 平台都一样：
kb-status

# LLM 会运行 python kb.py check
# 返回三级检查报告：
# A 级（基础）: 缺字段、死链接、语法错误
# B 级（内容）: 步骤不可执行、缺回滚
# C 级（全面）: 知识过期、孤立知识
#
# 对每个问题给出具体的修复建议
💡 建议： 每周运行一次 kb-status，保持知识库健康。

语义搜索 vs 关键词搜索

知识库现在使用 sqlite-vec 向量数据库，入库时自动为每条知识生成 AI 语义向量。搜索时不再需要匹配相同关键词。

对比体验

你搜什么关键词搜索（回退）语义搜索（默认）
"网站挂了怎么办" ❌ 找不到 — 知识里没有"挂"字 ✅ 找到"Nginx 故障恢复流程" (0.47)
"CI/CD 怎么配" ❌ 找不到 — 知识里写的是"Jenkins 部署" ✅ 找到"Jenkins 部署流程" (0.76)
"nginx 502 error" ⚠️ 低分 — "502"不在摘要里 ✅ 跨语言匹配 (0.70)
"磁盘满了" ⚠️ 低分 — 知识写的是"磁盘使用率超过 80%" ✅ 找到"磁盘空间清理"
💡 双模工作： 优先使用语义搜索。如果向量数据库未初始化，自动回退到关键词匹配。两者体验无缝。

重建向量索引

如果向量数据库丢失或损坏，可以从 wiki/*.md 文件重建：

# 重建全量向量索引
python kb.py rebuild-index

# 输出示例:
# 正在重建向量索引...
# ✓ 索引完成: 42 条知识
#   infra-daily: 12 条
#   infra-violation: 8 条
#   automation: 15 条
#   harbor: 7 条
💡 何时使用： 首次部署后、向量数据库文件丢失、或升级 embedding 模型后需要重新索引所有知识。

常见问题

Q: 上传文件后 Web 上看不到新知识？
A: 上传到 raw/ 只是存放原始文件。需要在 LLM 中使用 kb-ingest 完成导入，知识才会出现在 wiki/ 中。
Q: kb-search 返回空结果？
A: 检查 wiki/ 目录是否有 .md 文件。如果是空知识库，先用 kb-ingest 导入几条知识。
Q: LLM 说"SKILL 不可用"？
A: 确认 skills/ 目录已正确加载到 LLM 平台的 Skill 路径配置中。
Q: 知识页不显示流程图？
A: 流程图在导入时由 LLM 自动生成。如果导入时 LLM 不可用，则不会生成流程图。可以重新用 LLM 导入来补充。
Q: 如何更新已有知识？
A: 上传新版本的原始文件 → 用 kb-ingest 重新导入。Engine 会自动检测重复，生成 version+1 的新版本。
