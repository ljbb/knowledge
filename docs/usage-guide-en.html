DevOps & Infra Knowledge Base

One knowledge base, two interfaces — browse in your browser, drive with LLM Skills. Search "site is down" and find "Nginx Recovery Procedure" — no keyword matching needed.

Introduction

Built on the llm-wiki framework, this knowledge base flips the traditional RAG model: instead of retrieving raw documents at query time, the LLM incrementally builds and maintains a persistent, structured wiki of interlinked markdown files. The knowledge is compiled once, kept current, and served to both humans and LLMs.

🔮

Semantic Search

Search "site is down" → finds "Nginx Recovery Procedure". AI understands meaning, not just keywords.

🌐

Web Interface

Browse, search, upload source files. View complete knowledge entries with flowcharts rendered in the browser.

🤖

LLM Skills

Ingest, search, and health-check via agentskill.io standard Skills. Works with Claude Code, OpenCode, and GitHub Copilot.

📄

Markdown Core

Knowledge lives as Markdown files. Both humans and LLMs can read them directly. Git-powered version history.

✅

3-Tier QA

Level A (basics) · Level B (content) · Level C (comprehensive). Every entry is verified before ingestion.

Quick Start

  1. Initialize
    python kb.py init
  2. Configure at least one LLM API key
    ANTHROPIC_API_KEY=sk-ant-... or DEEPSEEK_API_KEY=sk-... or GEMINI_API_KEY=...
  3. Start the web server
    python kb.py serve → Open http://localhost:8000
  4. Load Skills into your LLM
    Point Claude Code / OpenCode / Copilot to the skills/ directory.

Knowledge Page — Read Details

Each knowledge page displays top to bottom:

  1. Metadata — ID, category, version, status, author, last updated
  2. Scenario — when and why to use this knowledge
  3. Steps — numbered instructions with executable commands
  4. Flowchart (if available) — Mermaid visualization of the process
  5. Verification — checklist to confirm success
  6. Rollback — recovery steps if something goes wrong
  7. Execution Interfaces — Harbor CLI, Jenkins, GitHub Action links
💡 When to use: You need to perform a specific operation. Open the knowledge page, follow the steps, and check off each verification item.

LLM Usage — How It Works

The knowledge base provides 3 meta Skills following the agentskill.io standard. They can be loaded into Claude Code, OpenCode, or GitHub Copilot. The LLM operates through Skills → CLI → Engine pipeline.

┌──────────┐     ┌──────────────┐     ┌──────────┐
│   You     │────▶│  LLM + Skill │────▶│  Engine  │
│ "ingest"  │     │ kb-ingest    │     │ ingest() │
└──────────┘     └──────────────┘     └──────────┘
                                             │
                              ┌──────────────┼──────────────┐
                              ▼              ▼              ▼
                         wiki/*.md     index.md       log.md
                         + .html       (updated)      (appended)
📥

kb-ingest

Ingest knowledge: receive text/files → LLM structures content → dedup → QA → write files → generate Skill

🔍

kb-search

Search knowledge: LLM extracts keywords → Engine executes search → returns matching Skill → LLM loads it

📊

kb-status

Health check: run 3-tier QA → summarize issues → provide fix recommendations per entry

Ingest Knowledge — Using kb-ingest

This is the only way to add knowledge. The flow is consistent across all LLM platforms:

# In Copilot Chat:
Use kb-ingest to import this document

[Paste content or drag in a file]

# Or specify a file already in raw/:
Ingest raw/nginx-restart-guide.md into the knowledge base
# In Claude Code:
> /kb-ingest

Claude will guide you through:
1. Provide knowledge content (paste or file path)
2. LLM auto-structures per template
3. LLM checks completeness, prompts for missing sections
4. Confirm and auto-ingest
# In OpenCode:
Please use the kb-ingest skill to import this content:

[Paste content]

OpenCode will call python kb.py ingest to complete the process.
  1. Provide source content — paste text or specify a file path in raw/
  2. LLM structures it — extracts scenario, steps, verification, rollback; fills in missing parts
  3. LLM writes to temp file — saves structured Frontmatter + Markdown as a temporary file
  4. CLI executes Pipeline — python kb.py ingest raw/doc.md --content /tmp/llm-output.md
  5. Full Pipeline runs — dedup → QA → write wiki → update index → generate Skill → build vector index

Read Knowledge — Using kb-search

Search is the entry point for LLMs to read knowledge. The LLM doesn't scan files — it delegates to the Engine.

# In Copilot Chat:
kb-search how to safely restart nginx in production

# Copilot will:
# 1. Call python kb.py search "nginx restart" --category infra-daily
# 2. Return matching knowledge entries with scores
# 3. Load the best-matching knowledge Skill
# 4. Present the complete step-by-step guide
# In Claude Code:
> /kb-search nginx restart procedure

Claude will search the KB and provide the operation guide.
# In OpenCode:
Use kb-search to find the disk cleanup procedure
Token-efficient design: Search returns only title + summary (~50 tokens per result). The full Skill (~500 tokens) is loaded only after you confirm the match. No need to load 200 entries into context.

Health Check — Using kb-status

Periodically have the LLM check knowledge base health:

# Works the same on any LLM platform:
kb-status

# The LLM runs python kb.py check and returns:

# Each issue comes with a specific fix recommendation

Semantic Search vs Keyword Search

The knowledge base now uses sqlite-vec vector database. Every ingested entry is automatically indexed with AI embeddings — no more keyword matching.

Comparison

You searchKeyword (fallback)Semantic (default)
"site is down" ❌ No match — no entry contains "down" ✅ Finds "Nginx Recovery Procedure" (0.47)
"how to set up CI/CD" ❌ No match — knowledge uses "Jenkins deploy" ✅ Finds "Jenkins Deployment" (0.76)
"disk almost full" ⚠️ Low score — entry says "usage exceeds 80%" ✅ Finds "Disk Cleanup"
"nginx 502 error" ⚠️ Low — "502" not in summary ✅ Cross-language match (0.70)
💡 Dual-mode: Semantic search runs by default. If the vector database is unavailable, it automatically falls back to keyword matching. The experience is seamless.

Rebuild Vector Index

If the vector database is lost or corrupted, rebuild it from wiki/*.md files:

# Rebuild the entire vector index
python kb.py rebuild-index

# Example output:
# Rebuilding vector index...
# ✓ Indexed: 42 entries
#   infra-daily: 12
#   infra-violation: 8
#   automation: 15
#   harbor: 7
💡 When to use: After initial deployment, if data/knowledge_vec.db is deleted, or after upgrading the embedding model.

Troubleshooting

Q: Uploaded a file but can't see it in the knowledge base?
A: Uploading to raw/ only stores the source file. You must use kb-ingest in an LLM to process and integrate it into wiki/.
Q: kb-search returns no results?
A: Check if any .md files exist in wiki/. If the KB is empty, use kb-ingest to add some knowledge first.
Q: LLM says "Skill not available"?
A: Verify the skills/ directory is correctly configured in your LLM platform's Skill path settings.
Q: Flowchart not showing on knowledge page?
A: Flowcharts are generated by the LLM during ingestion. If the LLM was unavailable during ingest, no flowchart is generated. Re-ingest with an LLM available to add one.
Q: How do I update existing knowledge?
A: Upload a new version of the source file → re-ingest with kb-ingest. The Engine auto-detects duplicates and creates version+1 while preserving the old version.
